Project Presentation:
The project is a game created with the aim of requiring a rollback system. In other words, action-oriented.
It was created during my 2nd year at the SAE in Geneva.

The project is based on the principles of CPP SOLID, where each class has a single Responsibility for rollback purposes.


It has the following structure:
- AudioManager, an Audio Manager used to play the in-game music, manage the volume, loop the music or mute it.
- Constant, a constant header shared throughout the project via a name space
- Events, Header filled with a structure for managing events
- Frame Inputs a class for managing inputs and their copies
- Game App the application that makes the link between the different logics
- Game Logic, the class that manages the world's physical actions.
- Image, a custom class that uses the raylib system to draw and adds the option of centring the pivot.
- Network Logic, manages all the network side of sending inputs to the client connection, using the Photon Realtime library.
- Player Manager, which manages the game play elements of the player as well as updating the player in the physical world.
- Raylib Wrapper, a wrapper for the raylib library which had redefinition conflicts with the Photon library.
- Renderer, to draw the different elements of the game in the raylib window.
- Rollback Manager, for rollback management, copying the validated physical world, resimulating the logic after a rollback.
